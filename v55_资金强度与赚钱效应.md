# V55 — 行业倾斜新信号源: 资金强度 + 赚钱效应（OHLCV 派生）

> 用户: 认可 V54 结论(行业趋势有价值但只能滞后), 提议试非趋势行业信号: 估值分位(满变量) / 情绪·资金(资金流入) / 行业赚钱效应改善。本篇先落地"可从现有数据算"的两条——资金强度、赚钱效应; 估值分位因缺基本面数据暂阻断。
> 承接 V54: 可交易趋势信号(avgdif / indmacd)均与个股 slope 同维; 真正正交需非趋势信号。本篇验证"资金/赚钱效应"是否构成真正的正交补充。

## 5. ⚠️ 关键诚实留痕: WALK 块全局 TILT_W bug 发现与修复

- **现象**: 首次 `WALK=1 EXP=signalv2` 跑出 avgdif k0.26 全样本 22.31、OOS 4.32, 与 V53/V54 验证的 32.30/5.33 严重不符。
- **根因**: `simulate()` 倾斜计算用**全局 `TILT_W`**(行607 `sc*=(1+TILT_W*clip(...))`), 而局部 `tilt=cfg['tilt']` 未被用于该处。主循环经 `_simulate_wrap` 设置全局 `TILT_W`, 但 **WALK 块直接调 `simulate()` 未走 wrapper**, 于是用了 GRID 循环残留的全局值。本次 `EXP=signalv2` 的 GRID 末位配置 tilt=0.40 → 所有 WALK 内倾斜调用都被当成 0.40 → 假值 22.31/4.32。
- **影响范围**: 仅影响 **WALK=1 且 EXP≠ablation** 的运行(V53 当时在 `EXP=ablation` 下 GRID 末位恰为 tilt=0.26, 残留值碰巧正确)。**V53 记录的 WF 数字(avgdif 5.33/3-4, consec 4.96/2-4)经本次修复后重跑完全吻合 → 仍然有效, 无需修正。**
- **修复**: `simulate()` 开头自设置 `TILT_W/BETA_W=cfg.get(...)`, 消除对残留全局值的依赖(行483-488)。修复后重跑: avgdif k0.26 全样本=32.30、OOS=5.33(3/4), consec OOS=4.96(2/4) —— 与 V53 一致。
- **教训**: 任何 WALK 运行现在在任意 EXP 下都可靠; 此前若在 `EXP=signal`/`signalv2` 下带 `WALK=1` 跑过, 其倾斜类 OOS 数字无效(本次已纠正)。

## 6. 综合结论

1. **资金强度(amt/amtr) 与 赚钱效应(breadth/breadth_d) 作为独立行业倾斜信号源, 不成立**: 全样本(22~25)与 OOS(3.83~3.96) 均弱于 avgdif(32.30 / 5.33), 且 OOS 低于无倾斜基线(4.54)。
2. **它们确实比 avgdif 更正交于斜率**(corr(slope) 0.07 vs 0.149), 也非 avgdif 冗余(正交性诊断证实独立信息)。但**正交 ≠ 预测力强**——这两个维度携带独立但弱的信号。
3. **印证 V54 判断**: 资金/赚钱效应仍是价格/成交量派生, 同属滞后域, 不是估值那样的"状态量"正交突破。它们补充的是"参与广度/活跃强度"切面, 但该切面在 A股中证1000 样本内预测力不足。
4. **组合可能性(低优先)**: 因新信号与 avgdif 正交, 理论上可叠加 `z_avgdif + λ·z_breadth` 不双重计数。但新信号 OOS(3.8~4.0) 已低于基线, 叠加大概率稀释 avgdif 的 5.33, 增益有限; 不优先。
5. **估值分位经用户判定为慢变量(对短周期倾斜无用), 已不纳入**。故在当前 OHLCV 短周期数据域内, **不存在理论上的"状态量正交"行业信号源**; 行业信号源搜索已穷尽(见下)。
