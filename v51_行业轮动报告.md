## 1. 思路：轮动 vs 过滤的本质区别

| 维度 | 行业共振「硬过滤」(V49/V50 已证伪) | 行业轮动「软倾斜」(本次) |
|---|---|---|
| 对候选的处理 | 弱行业直接 `continue` 剔除 | 不删，只调整排名/权重 |
| 资金去向 | 弱势期资金空置（过度 pruning） | 资金始终在场，向强势行业集中 |
| 信号 | 行业 avgDIF 是否 > 阈值（绝对） | 行业动量横截面对比（谁更强） |
| V50 结论 | 量比基线上无增益 | 待测 |

核心判断：V46 真实 alpha 是**量比**；个股 DIF% 与行业 avgDIF% 同源，所以"硬过滤"无效。但**软倾斜**不改变成交数量、只在满仓时决定"买谁"，理论上能在不砍交易的前提下改善持仓结构。

---

## 2. 三类轮动机制

1. **TILT 软加权**：`score *= 1 + k·z(行业动量)`。动量信号三选一——行业 avgDIF 横截面 z 值 / 行业平均量比 / 行业金叉广度（breadth）。
2. **SLOT 配额轮动**：把 N 个仓位按行业动量 softmax 分配，强势行业占更多名额，余量溢出。
3. **对照**：无轮动基线（精确复现 +557.79% / 19.43% / 28.71）。

## 4. 核心结论

1. 🟢 **行业轮动（软倾斜）有效（定性保留），但"强于 consec"不成立（V53 推翻）。**
   纯量比基线上，行业 avgDIF 软倾斜 k≈0.26 → 比率 **32.30**（收益 563.73% / 回撤 17.45%），**低于 consec.py 的 33.85**。即行业动量倾斜是 modest 的辅助层，并非能碾压 consec 三因子的 dominant alpha（详见 V53 消融）。

2. 🔴 **倾斜与 consec 不叠加。**
   把同款倾斜加在 consec 全因子上，比率从 33.85 **降到 31.52**。二者信息重叠 + 叠加导致过度集中 → 应选「量比基线 + 行业倾斜」，而非「consec + 倾斜」。

3. 🔴 **SLOT 硬配额轮动失败**（比率 14~18）。
   强制按行业分配名额削弱分散度，余量溢出也救不回。纯轮动分配不如软倾斜。

4. ⚠️ **k 敏感但结论稳健**：局部峰值 0.26，邻点均 >30 且全胜基线 28.71。具体 k 属样本内优化，不过拟合风险可控。

5. ✅ **口径校验通过**：本引擎同时精确复现 `final.py`（557.79）与 `consec.py`（596.94），结果可信。

## 5. 与 V49/V50 的承接

- V50 证伪「行业共振硬过滤」（量比基线上无增益：过度 pruning 或 no-op）。
- V51 证明「行业轮动软倾斜」才是正确用法：同是行业动量信号，**软倾斜（选股）有效、硬过滤（剔除）无效**。机制区分明确。
- ⚠️ **最优策略以 V53 为准**：全样本最优是 **consec全因子（无倾斜）= 596.94% / 17.63% / 33.85**；量比+倾斜(k0.26) = 563.73% / 17.45% / 32.30（低于 consec）。原"最优=量比+倾斜 651.82/36.77"已作废。

## 6. 下一步候选

- 对倾斜 k 做 walk-forward / 分段稳定性检验（防过拟合）。
- 试行业动量用「行业指数自身 MACD」或「前瞻收益」作信号，而非个股 avgDIF 汇总。
- 与 consec 的「连涨减分」单独组合（倾斜已含动量，连涨减分可能仍有边际价值）。
